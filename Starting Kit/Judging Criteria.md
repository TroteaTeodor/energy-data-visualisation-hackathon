# Energy Data Visualisation Hackathon — Brussels, 13 June 2026

## Jury Document

# Judging Criteria

**Total score per project: 25 pts**

## 1. Visual Clarity & Communication — 5 pts

Does the visualisation make a complex energy concept immediately understandable to a non-technical Brussels household? A great score means someone with no data background can look at the output and genuinely understand something new about their energy bill, consumption, or the grid.

| Score | Criteria |
| --- | --- |
| 5 | Instantly clear, zero jargon — could be shared on social media as-is |
| 3–4 | Clear with minor friction — small tweaks would make it self-explanatory |
| 1–2 | Requires explanation to be understood |
| 0 | Confusing or inaccessible to a non-technical audience |

## 2. Real-World Impact & Relevance — 5 pts

Does the solution address a real, documented barrier to energy transition in Brussels? Is there a credible path to putting it in front of actual households, cooperatives, or policymakers? Bonus weight if the solution directly serves the most vulnerable: renters, low-income households, and people experiencing energy poverty.

| Score | Criteria |
| --- | --- |
| 5 | Tackles a specific, documented problem — deployment path is obvious |
| 3–4 | Real problem, plausible impact — path to deployment needs work |
| 1–2 | Interesting but tangential — unclear who benefits or how |
| 0 | No clear real-world application |

## 3. Data Integrity & Honesty — 5 pts

Are data sources cited and reliable? Are comparisons fair and complete — not cherry-picked? Does the visualisation avoid creating fear or misleading impressions? This criterion is specific to this hackathon's mission: a tool that oversimplifies or distorts, even unintentionally, scores low here.

| Score | Criteria |
| --- | --- |
| 5 | All data sourced, comparisons honest, uncertainty shown where relevant |
| 3–4 | Mostly sound — minor gaps in sourcing or framing |
| 1–2 | Some misleading elements or unsourced claims |
| 0 | Data misrepresented or fabricated |

## 4. Technical Execution — 5 pts

Is the prototype live and demonstrable? Does it work on real data, not mock data? Judges assess what was actually built in one day — ambition is valued, but a working MVP beats an impressive mockup with no data behind it.

| Score | Criteria |
| --- | --- |
| 5 | Fully working demo on real data — robust and interactive |
| 3–4 | Working demo with some limitations — real data used |
| 1–2 | Partial prototype or mockup — limited interactivity |
| 0 | No working demo |

## 5. Presentation & Storytelling — 5 pts

Did the team clearly explain the problem they chose, show their solution, and run a live demo within 5 minutes? Judges assess whether a non-expert in the room understood what was built and why it matters — not just whether the slides were polished.

| Score | Criteria |
| --- | --- |
| 5 | Problem, solution, and demo delivered crisply — compelling and clear |
| 3–4 | Good structure — minor clarity issues or slight time overrun |
| 1–2 | Hard to follow — demo missing or unclear |
| 0 | No coherent presentation |

## Note to the Jury

Dimension 3, Data Integrity, is weighted equally to the others — this is intentional. This hackathon's core mission is to make energy data trustworthy, not just beautiful. A visually stunning tool that misleads cannot win.
